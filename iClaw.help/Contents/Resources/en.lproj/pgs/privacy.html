iClaw Help

Privacy & Security

On-device AI

By default, iClaw uses Apple Intelligence (Foundation Models) for all AI processing. Your conversations, queries, and personal data are processed entirely on your Mac. No data is sent to Anthropic, OpenAI, or any other AI service.

Ollama (optional)

iClaw can optionally use Ollama as an alternative AI backend. Ollama runs open-source language models locally on your Mac — just like Apple Intelligence, all processing stays on-device.

When Ollama is selected in Settings > Models:

  • iClaw communicates with the Ollama server at localhost:11434
  • No data leaves your machine — Ollama runs entirely locally
  • iClaw auto-detects installed models and prefers models already loaded in memory
  • If Ollama is not installed or not running, the option is disabled in Settings

You can switch between Apple Intelligence and Ollama at any time in Settings > Models.

Network access

iClaw connects to the internet only when tools need external data:

  • Weather data from weather services
  • Stock quotes from financial APIs
  • Web search results and page fetching
  • Currency exchange rates
  • News headlines
  • Skill API calls (custom skills are sandboxed to HTTP only)

AI inference never uses the network — whether you use Apple Intelligence or Ollama.

Local storage

Conversation history and long-term memory are stored in a local SQLite database on your Mac. Nothing is synced to the cloud.

Permissions

iClaw requests system permissions only when a tool needs them:

  • Location — Weather and location-aware queries
  • Contacts — Contact lookups and messaging
  • Calendar & Reminders — Event and task management
  • Microphone — Voice input and audio transcription
  • Screen Capture — Analyzing on-screen content
  • Automation (Apple Events) — Controlling other apps

You can review and manage all permissions in Settings > Permissions or in System Settings > Privacy & Security.

Action consent

Tools that perform actions (sending messages, creating events, running shortcuts) have a consent policy:

  • Safe — Read-only actions, no confirmation needed
  • Requires Consent — Asks before acting (can be auto-approved in Settings)
  • Destructive — Always requires explicit confirmation
